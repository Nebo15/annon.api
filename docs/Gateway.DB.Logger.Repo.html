    gateway v0.1.83 Gateway.DB.Logger.Repo

    This repo is used to store request and responses.

    We recommend database with low write latency, because it will have up to 2 writes for each API call that is going trough Annon.

    Also, if Idempotency plug is enabled and X-Idempotency-Key: <key> header is sent by a consumer, you can expect an additional read request.

    page(queryable, paging, opts \\ []) 
    page(queryable :: Ecto.Query.t, paging :: Ecto.Paging.t, opts :: Keyword.t) ::
  {[Ecto.Schema.t], Ecto.Paging.t} |
  no_return

    Fetches all entries from the data store matching the given query with applied Pagging.

    May raise Ecto.QueryError if query validation fails.

    Options

    See the “Shared options” section at the Ecto.Repo module documentation.

    Example 

    # Fetch 50 post titles
query = from p in Post,
     select: p.title

MyRepo.all(query, %Ecto.Paging{limit: 50})
